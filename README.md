# ThreadsAI — Threads運用プロジェクト

Threadsアカウントの運用を効率化するためのプロジェクトです。
投稿管理・競合分析・アイデア整理をまとめて行います。

---

## フォルダ構成

```
ThreadsAI/
├── competitors/   競合・参考アカウントの分析
├── posts/         投稿済み・予定投稿のテキスト管理
├── ideas/         投稿アイデアのメモ
├── analysis/      エンゲージメントやトレンドの分析結果
└── prompts/       Claude等AIへの指示テンプレート
```

---

## 各フォルダの使い方

### `competitors/`
参考にしたいアカウントや競合の投稿をまとめます。  
ファイル例: `競合アカウント名.md` に投稿傾向・フォロワー数・特徴を記録。

### `posts/`
実際に投稿するテキストを管理します。  
- `YYYY-MM-DD_タイトル.md` の形式で保存するとわかりやすいです。  
- 投稿済みのものは先頭に `[済]` をつけると管理しやすくなります。

### `ideas/`
思いついたネタや企画をストックします。  
細かく整理しなくてよいので、気軽にメモを追加してください。

### `analysis/`
投稿のインプレッション・いいね・フォロワー推移などをまとめます。  
週次・月次のレポートをここに保存することを推奨します。

### `prompts/`
投稿文の生成・リライト・分析に使うAIへの指示文を保存します。  
再利用できるプロンプトをテンプレート化しておくと効率的です。

---

## 運用フロー例

1. **アイデア出し** → `ideas/` にメモ
2. **投稿文作成** → `prompts/` のテンプレートを使ってAIで生成
3. **投稿管理** → `posts/` に保存して投稿・管理
4. **振り返り** → `analysis/` にエンゲージメントを記録
5. **競合チェック** → `competitors/` に参考事例を蓄積

---

## Tips

- Claude Codeに「`posts/`の最新投稿を参考に新しい投稿を3つ作って」のように依頼すると、過去の文体を活かした投稿生成ができます。
- `analysis/` のデータをもとに「バズった投稿の共通点を分析して」と依頼するのも効果的です。
